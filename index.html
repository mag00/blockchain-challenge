About

The blockchain challenge is a very hackable game. If you've done the most computational work, you become the leader. That's it.

But, there are ways to disrupt someone else's work...

Every week the leader changes. It's inspired by the concept of "difficulty" in Bitcoin, and "The Button" by Reddit.

The game is based on beating the previous difficulty, and a "world record" is stored based on who has collided (so to speak) the hardest. This world record persists over the course of weeks and stores the account of the record holder.

When you become the leader, you receive any funds in the contract. You also control the message stored in the contract so you can tell us all to vote for Trump.

The contract publishes events so you can watch people play in the Mist client.

How To Play

The contract accepts a string in function Overthrow. The string will be SHA3 hashed and XOR'd with the current LeaderHash. If the result is less than the Difficulty, your Ethereum address will be stored as the new leader, and the difficulty will equal that amount of effort. You'll then have access to the Change Leader Message function in the contract, as it is gated by the leader's address.

Strategy

I suspect that strange strategies will arise from this game. Here were my observations while creating it:

  1. You can submit "small" collisions to reset anyone trying a larger collision
  2. You can rush a large collision to prevent small-timers from trying strategy #1 out
  3. You can wait until less people are playing, like "The Button" at reddit
  4. You can simply not make any attempts and avoid pushing the difficulty up, like in an auction

There are others, but I'd like to see some hackers figure it out. Read the source.

Abuse / Problems / Bugs

The contract has an admin selfdestruct() in case a bug makes the game no fun or some kind of abuse breaks it down. Submit a pull request if you think something is broken. I can change this page to reflect the latest game.
